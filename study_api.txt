작업폴더 생성.
npm install -g yarn 

yarn init -y 

package.json에서 
{
  "name": "APIstudy",
  "version": "1.0.0",
  "main": "index.js",
  "license": "MIT",
  "private": true,
  "workspaces": [
    "client",
    "server"
  ],

  "scripts": {
    "client": "yarn workspace client start",
    "server": "yarn workspace server start"
  }
}
workspaces 설정하면 yarn intall 할때 여러 프로젝트에서 중복으로 모듈을 설치하는 것을 방지해줌. 
즉 client 폴더와 server 폴더에서 각각 package.json 에 있는 모듈을 설치할때 중복을 제거해줌.
